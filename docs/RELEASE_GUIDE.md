## 二、Cocos Creator 操作步骤

### 1. 预览测试

1. 在 Cocos Creator 中打开项目（已打开）
2. 点击编辑器顶部的 **▶ 预览** 按钮
3. 选择浏览器预览
4. 检查控制台是否有错误

### 2. 构建微信小游戏

1. 点击菜单 **项目 → 构建发布**
2. 选择发布平台：**微信小游戏**
3. 填写构建配置：
   - AppID: `wxd57db417d5bd7fa1`
   - 设备方向: `横屏`
   - 资源服务器地址: (留空，使用本地资源)
4. 点击 **构建**
5. 构建完成后点击 **运行**

### 3. 微信开发者工具

1. 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 使用微信扫码登录
3. 导入项目，选择构建目录：`build/wechatgame/`
4. 在开发者工具中预览和调试

## 三、真机测试

### 1. 扫码预览

### 2. 真机调试

## 四、资源服务器配置（可选）

如果游戏包体较大，可以使用远程资源服务器：

### 1. 上传资源
将 `build/wechatgame/res/` 目录上传到CDN服务器

### 2. 配置服务器地址
在构建设置中填写资源服务器地址，例如：
```
https://your-cdn.com/heros/res/
```

### 3. 分包加载
如果首包超过4MB，需要配置分包：
- 在 `project.json` 中配置 `subpackages`
- 将部分资源放入分包目录

## 五、提交审核

### 1. 准备材料
- 游戏图标: 512×512 PNG
- 游戏截图: 5张 750×1334 或 1920×1080
- 游戏介绍文案
- 隐私协议链接
- 软件著作权证明（如需要）

### 2. 上传代码
在微信开发者工具中点击 **上传**，填写版本号和备注

### 3. 提交审核
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入 **管理 → 版本管理**
3. 点击 **提交审核**
4. 填写游戏信息

### 4. 审核周期
- 一般 1-3 个工作日
- 首次提交可能需要更长时间

## 七、常见问题

### Q: 构建失败
检查 TypeScript 编译错误，查看控制台输出

### Q: 黑屏/白屏
检查 Game.ts 是否正确初始化，检查场景配置

### Q: 包体过大
- 压缩纹理资源
- 使用远程资源
- 配置分包加载

### Q: 真机性能差
- 减少DrawCall
- 使用对象池
- 优化渲染逻辑

## 八、项目文件结构

```
heros/
├── assets/
│   ├── scenes/           # 场景文件
│   │   ├── MainMenu.scene  # 启动场景
│   │   ├── Town.scene      # 主城场景
│   │   └── Battle.scene    # 战斗场景

│   ├── resources/        # 动态资源
│   └── textures/         # 静态贴图

├── build/                # 构建输出
│   └── wechatgame/       # 微信小游戏
└── scripts/              # 工具脚本
    ├── bind-prefab-scripts.js
    └── bind-game-to-scene.js
```
